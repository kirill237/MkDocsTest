# Общие требования и ограничения

### Требования к сервисам:
{--Сервис разрабатывается с учётом общих требований, приведённых в разделе Общие требования и ограничения.--}
{++text++}
При возникновении ошибок должно происходить логирование с описанием ошибки (на латинице). При возникновении исключительной ситуации логировать как можно больше информации о ситуации, приведшей к ошибке.  
При возникновении ошибки в работе сервиса УС должен выполнить действия, описанные в статье _omni-cc-orders_. Требования к ошибкам.
Валидация на обязательность входных параметров должна выполняться с помощью библиотеки  **omni-web-starter** :  
* Если хотя бы одна из проверок не пройдена, то УС записывает в лог исключительную ситуацию с уровнем ERROR и завершает свою работу с параметрами, определёнными библиотекой, где _error/id_ == _^^iek2lo^^_ . При этом УС переопределяет error/code значением badRequest.
